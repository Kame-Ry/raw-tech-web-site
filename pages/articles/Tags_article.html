Interactive Tags Setup: A Novice IT Technician's Journey

Interactive Tags Setup is a dynamic approach to filtering content on a projects page. As a novice IT technician diving into the world of JavaScript and CSS, this project was a blend of trials, errors, and learning curves. In this article, I'll share my journey, the challenges I faced, and the improvements I plan to make in future iterations.

The Setup: Files in Play

The solution revolves around three core files:

  • Tags.js – This JavaScript file manages tag generation, event listeners, and filtering logic.
  • post-entry.css – The CSS file ensures that project cards and tags have a sleek, modern look.
  • projects.html – Acts as the container for the interactive elements and loads the required scripts.

The Hurdles and How I Overcame Them

While the idea was straightforward, I encountered several issues along the way:

  • Delayed Post Loading: Initially, the tag generation function was running before the project entries were fully loaded. I implemented a polling mechanism using the waitForPosts() function that checks every 500ms for .post-entry elements.
  • URL Parameter Handling: Ensuring the page maintained the state of a selected tag through URL parameters was a bit tricky. By parsing the URL on page load, I could apply filters dynamically, making the user experience more consistent.
  • Active State Management: It was essential to accurately reflect the selected tag by toggling the active class. I addressed this by comparing clicked tags with the URL parameter and updating the display accordingly.

Steps Taken to Implement the Feature

  1. Set up an event listener for DOMContentLoaded to ensure all resources were loaded.
  2. Implemented URL parameter parsing to retrieve and apply a pre-selected tag.
  3. Developed the waitForPosts() function to delay execution until project entries were present.
  4. Created the generateTagList() function to extract unique tags from project metadata and render them dynamically.
  5. Built the filterProjects() function to show or hide projects based on the selected tag.
  6. Added a document-level click handler to manage tag selection and deselection, redirecting the user appropriately.

Lessons Learned and Future Improvements

This project taught me the importance of timing in JavaScript execution and the value of clean, modular code. Some improvements I plan to implement include:

  • Event-Driven Updates: Instead of polling for post elements, I’d like to explore event-driven solutions that could make the code more efficient.
  • Error Handling: Incorporating better error messages to inform the user when no tags or posts are available.
  • Code Refactoring: Further refactoring the JavaScript code to enhance readability and maintainability.

Conclusion

Setting up interactive tags for dynamic project filtering has been a rewarding experience. Despite some initial challenges, the final implementation not only enhanced the user experience but also taught me valuable lessons in asynchronous programming and UI state management. I look forward to refining this feature further, making it even more robust and user-friendly in future projects.
